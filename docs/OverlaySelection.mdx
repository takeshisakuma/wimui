# オーバーレイコンポーネントの使い分け

UIの目的に応じて、最適なオーバーレイを選択するためのガイドラインです。

## 情報設計の原則

### モーダル (Modal) とモードレス (Modeless) の使い分け

<table>
  <thead>
    <tr>
      <th>種類</th>
      <th>動作</th>
      <th>判断のポイント</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>モーダル</strong></td>
      <td>背後の画面を操作させない。</td>
      <td><strong>タスクに集中させたい、または致命的なミスを防ぎたいとき。</strong>（Dialogなど）</td>
    </tr>
    <tr>
      <td><strong>モードレス</strong></td>
      <td>表示中も背後の画面を操作できる。</td>
      <td><strong>元の画面を見ながら作業を続けさせたいとき。</strong>（Popover, 一部のDrawerなど）</td>
    </tr>
  </tbody>
</table>

### モバイルへの配慮

<table>
  <thead>
    <tr>
      <th>デバイス</th>
      <th>推奨コンポーネント</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>デスクトップ</strong></td>
      <td>情報量が多い場合は **Drawer**（サイドパネル）が視認性が高く、操作もしやすいです。</td>
    </tr>
    <tr>
      <td><strong>モバイル</strong></td>
      <td>画面下部から出る **BottomSheet** が片手操作に最も適しています。Dialog は入力要素が多いとキーボードで見えなくなるため注意が必要です。</td>
    </tr>
  </tbody>
</table>
