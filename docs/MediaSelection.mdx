# <T k="guide_media_title" />

画像、ビデオ、オーディオを効果的に使用して、視覚的・聴覚的に情報を伝えます。

## ラインナップ

<table>
  <thead>
    <tr>
      <th>コンポーネント</th>
      <th>主な用途</th>
      <th>活用のヒント</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>Image</strong></td>
      <td>画像表示</td>
      <td>`lazy loading` を活用してパフォーマンスを最適化し、`AspectRatio` コンポーネントでレイアウトシフトを防止します。</td>
    </tr>
    <tr>
      <td><strong>Video / Audio</strong></td>
      <td>マルチメディア再生</td>
      <td>カスタムコントロールを付与し、アプリのトーン＆マナーに合わせます。</td>
    </tr>
    <tr>
      <td><strong>Carousel</strong></td>
      <td>複数要素の閲覧</td>
      <td>バナー広告、製品ギャラリー、ストーリー形式のコンテンツ表示に最適。</td>
    </tr>
  </tbody>
</table>

## ユーザビリティとパフォーマンス

<table>
  <thead>
    <tr>
      <th>項目</th>
      <th>ガイドライン</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>代替テキスト (Alt)</strong></td>
      <td>画像やビデオには必ず代替テキストを設定してください。視覚障害を持つユーザーだけでなく、SEO にも寄与します。</td>
    </tr>
    <tr>
      <td><strong>自動再生の制限</strong></td>
      <td>音声やビデオの自動再生はユーザーを驚かせ、アクセシビリティ標準に反する場合が多いため、必要最小限（ミュート状態での背景動画など）に留めてください。</td>
    </tr>
    <tr>
      <td><strong>スケルトンの活用</strong></td>
      <td>重いメディアファイルの読み込み中は、`Skeleton` コンポーネントを表示して、ユーザーに進行状況を伝えてください。</td>
    </tr>
  </tbody>
</table>
