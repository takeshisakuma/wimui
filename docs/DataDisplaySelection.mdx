# データ表示コンポーネントの使い分け

スクロール、チャット、動的なコンテンツの表示に特化したコンポーネントです。

## コンポーネント比較

<table>
  <thead>
    <tr>
      <th>コンポーネント</th>
      <th>目的</th>
      <th>主な用途</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>InfiniteScroll</strong></td>
      <td>無限スクロール</td>
      <td>ページ最下部に到達すると自動で次のデータを追加読み込みする。SNSフィード、商品一覧など、ページネーションを使わず連続してデータを表示する場合。</td>
    </tr>
    <tr>
      <td><strong>Markdown</strong></td>
      <td>Markdownレンダリング</td>
      <td>Markdown記法の文字列を受け取り、適切なHTML要素（見出し、コードブロック、リストなど）として整形して表示する。ブログ記事、ドキュメント、コメント欄に最適。</td>
    </tr>
    <tr>
      <td><strong>ChatUI</strong></td>
      <td>チャットインターフェース</td>
      <td>メッセージの送受信を表示するチャット型のUIコンポーネント群（ChatContainer / ChatMessageList / ChatMessage / ChatInput / ChatAvatar）。サポートチャット、AIチャットボット、リアルタイムメッセージ画面に最適。</td>
    </tr>
  </tbody>
</table>

---

## 使い分けのポイント

<table>
  <thead>
    <tr>
      <th>項目</th>
      <th>ガイドライン</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>パフォーマンス</strong></td>
      <td>`InfiniteScroll` は大量データの逐次読み込みに有効ですが、仮想スクロールが必要なほどデータが膨大な場合は `DataGrid` の使用を検討してください。</td>
    </tr>
    <tr>
      <td><strong>セキュリティ</strong></td>
      <td>`Markdown` はユーザー入力のMarkdownをレンダリングする際、XSSリスクを防ぐためにサニタイズ処理を必ず適用してください。</td>
    </tr>
    <tr>
      <td><strong>リアルタイム更新</strong></td>
      <td>`ChatUI` をリアルタイムに更新する場合は、WebSocketや SSE（Server-Sent Events）との組み合わせを検討してください。</td>
    </tr>
  </tbody>
</table>
