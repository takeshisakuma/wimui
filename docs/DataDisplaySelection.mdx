# データ表示コンポーネントの使い分け

情報をユーザーに提示する際、その構造やユーザーが行いたい操作に応じて形式を選択します。

## 情報設計の原則

### 情報の優先順位付け

<table>
  <thead>
    <tr>
      <th>ポイント</th>
      <th>ガイドライン</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>重要な情報の強調</strong></td>
      <td>最も重要なデータはフォントサイズや色の重みを変えて、最初に目が向くようにします。</td>
    </tr>
    <tr>
      <td><strong>スキャン性の確保</strong></td>
      <td>ユーザーは情報のすべてを読むのではなく「スキャン（流し読み）」します。見出しや一貫した並び順が重要です。</td>
    </tr>
  </tbody>
</table>

### 状態への配慮

<table>
  <thead>
    <tr>
      <th>状態</th>
      <th>ガイドライン</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>空の状態 (EmptyState)</strong></td>
      <td>表示データがない場合は、ただの空白ではなく「データがありません」というメッセージと、次に何をすべきか（作成ボタンなど）を提示します。</td>
    </tr>
    <tr>
      <td><strong>読み込み中 (Loading)</strong></td>
      <td>データの取得に時間がかかる場合は、Skeleton や Spinner を使用して、ユーザーに進行中であることを伝えます。</td>
    </tr>
  </tbody>
</table>
