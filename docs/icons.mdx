# Icons

アプリケーション全体で使用するアイコンの設計指針と使い分けについてのガイドです。

具体的な `Icon` コンポーネントの使用方法や、利用可能なアイコンの一覧については、[Icon コンポーネントのドキュメント](?path=/docs/component-typography-icons-icon--docs) を参照してください。

## アイコンの使い分け指針

### 線 (Outline) が向いているケース

    <tr>
      <td>ナビゲーションメニュー(未選択時)（サイドバー・タブバー）</td> 
      <td> 全てが「塗り」だと画面が重苦しくなるが、「線」にすることでコンテンツ（中身）を主役にできまる<br/>
      挙動: 普段は「線」、クリックして選択されたら「塗り」に切り替える</td>
    </tr>
    <tr>
      <td>アイコンが密集する場所</td>
      <td>例：ツールバーなど</td>
    </tr>
        <tr>
      <td>情報の密度が高い場所</td>
      <td>ツールバーのようにアイコンが横並びで大量に並ぶとき、「塗り」だと視覚的なノイズ(黒い塊)が多くなりすぎて疲れるが、「線」ならスッキリと整列して見える<br/>
      例：B2Bツールや管理画面、デザインツールやドキュメント作成ソフトの操作パネル</td>
    </tr>
    <tr>
      <td>カードコンポーネント内の補助情報</td>
      <td>カード内の主役の「画像」や「タイトル」の邪魔をせず、必要な時だけ認識できる控えめな表現にする<br/>
      「お気に入り（ハート）」や「共有（シェア）」、「時間（時計）」など</td>
    </tr>
    <tr>
      <td>入力フォーム（Input Field）内の装飾</td>
      <td>アイコンが目立ちすぎると入力の邪魔になるので、細い線で添える<br/>
      例：検索バーの「虫眼鏡」や、パスワード入力欄の「目（表示/非表示）」アイコン</td>
    </tr>
  </tbody>
</table>

### 塗りつぶし (Filled) が向いているケース
<table>
  <thead>
    <tr>
      <th>場所</th>
      <th></th>

    </tr>
  </thead>
  <tbody>
    <tr>
      <td>選択状態(Active)の表現</td>
      <td>例：タブバーで今開いている項目</td>
    </tr>
    <tr>
      <td>重要なアクションボタン</td>
      <td>例：削除、保存、追加など</td>
    </tr>
    <tr>
      <td>複雑な背景の上</td>
      <td></td>
    </tr>
    <tr>
      <td>非常に小さいサイズでの表示</td>
      <td></td>
    </tr> 
    <tr>
      <td>強いメッセージを持つコンポーネント</td>
      <td>例：「警告（!）」や「エラー（×）」</td>
    </tr>
    <tr>
      <td>ボタン内のアイコン</td>
      <td>視覚的なバランスを取りやすい<br/>
      ボタンの背景色に埋もれない<br/>
      シルエットがはっきりするので、一目でアイコンの意味を理解できる</td>
    </tr>
    <tr>
      <td>テキストの頭に配置するアイコン</td>
      <td>視覚的なバランスを取りやすい<br/>
      主役であるテキストの邪魔をしない、「補助役」のため控えめな表現が適している<br/>
      シルエットがはっきりしているため、テキストと並べても埋もれにくく、アイコンの存在を認識しやすい<br/>
      テキストが太字（Bold）の場合: 塗りにすると文字の強さに負けない<br/>
      テキストが標準(Regular)の場合:塗りにすると、「箇条書きのバレット」のように、視線を誘導するガイドとして機能する</td>
    </tr>
  </tbody>
</table>
