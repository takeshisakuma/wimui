# ボタンコンポーネントの使い分け

ユーザーのアクションを促すボタンは、その重要度や役割に応じて視覚的な重み付けを使い分ける必要があります。

## Link vs Button

セマンティクス（意味付け）とアクセシビリティを正しく保つために、見た目ではなく「目的」で使い分けます。
詳細な使い分け基準については、専用のガイドを参照してください。

> 📘 **Guide**: **[Link vs Button (リンクとボタンの使い分け)](?path=/docs/guides-link-vs-button--docs)**

## 役割と優先度

<table>
  <thead>
    <tr>
      <th>種類 (Priority)</th>
      <th>視覚的な重み</th>
      <th>用途</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>Primary</strong></td>
      <td>強 (塗りつぶし)</td>
      <td>その画面で最も重要な単一のアクション（送信、保存、完了）</td>
    </tr>
    <tr>
      <td><strong>Secondary</strong></td>
      <td>中 (アウトライン)</td>
      <td>主要アクションを補完するアクション（キャンセル、編集、戻る）</td>
    </tr>
    <tr>
      <td><strong>Tertiary (Ghost)</strong></td>
      <td>弱 (テキストのみ)</td>
      <td>重要度の低いオプション、多用しても画面を圧搬しない</td>
    </tr>
    <tr>
      <td><strong>Danger</strong></td>
      <td>強 (赤/警告色)</td>
      <td>取り返しのつかない操作、注意が必要な操作（削除、退会）</td>
    </tr>
  </tbody>
</table>

## 特殊なボタン

<table>
  <thead>
    <tr>
      <th>コンポーネント</th>
      <th>概要</th>
      <th>いつ使うか (ユースケース)</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>Fab</strong> <br/><small>(Floating Action Button)</small></td>
      <td>画面の右下などに浮いているボタン。</td>
      <td>画面内のどこからでもアクセスしたい、そのページで最も頻繁に行われるアクション。</td>
    </tr>
    <tr>
      <td><strong>Button Group</strong></td>
      <td>複数のボタンを繋げて表示するコンポーネント。</td>
      <td>関連性の高い一連のアクションをまとめるとき（例：ソート順の切り替え）。</td>
    </tr>
    <tr>
      <td><strong>Copy Button</strong></td>
      <td>値をクリップボードにコピーするための専用ボタン。</td>
      <td>ユーザーが手作業で選択してコピーするのが難しい値の提供。</td>
    </tr>
  </tbody>
</table>

## ガイドライン

<table>
  <thead>
    <tr>
      <th>原則</th>
      <th>具体的なガイドライン</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>一貫性</strong></td>
      <td>同じアクションには、常に同じ種類（Priority）のボタンを使用してください。</td>
    </tr>
    <tr>
      <td><strong>配置</strong></td>
      <td>主要なアクション（Primary）は、一般的な視線移動の終点（例：右下）に配置します。</td>
    </tr>
    <tr>
      <td><strong>配置の競合</strong></td>
      <td>1つの画面（またはモーダル）に Primary ボタンは原則として1つだけに留めます。</td>
    </tr>
  </tbody>
</table>
