# ナビゲーションコンポーネントの使い分け

ユーザーがアプリケーション内を移動したり、表示内容を切り替えたりするための最適な手段を選択します。

## ナビゲーション比較表

<table>
  <thead>
    <tr>
      <th>コンポーネント</th>
      <th>配置</th>
      <th>階層レベル</th>
      <th>主な用途</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>Navbar</strong></td>
      <td>画面上部</td>
      <td>最上位</td>
      <td>アプリ全体の主要なセクション切り替え</td>
    </tr>
    <tr>
      <td><strong>Sidebar</strong></td>
      <td>画面横</td>
      <td>最上位〜中</td>
      <td>機能数が多い管理画面、複雑な階層構造</td>
    </tr>
    <tr>
      <td><strong>Tab</strong></td>
      <td>コンテンツ内</td>
      <td>中〜下</td>
      <td>同一コンテキスト内での表示切り替え</td>
    </tr>
    <tr>
      <td><strong>Breadcrumb</strong></td>
      <td>コンテンツ上部</td>
      <td>全レベル</td>
      <td>現在地の把握、上位階層への素早い復帰</td>
    </tr>
    <tr>
      <td><strong>Pagination</strong></td>
      <td>リスト下部</td>
      <td>リスト</td>
      <td>大量データのページ分割移動</td>
    </tr>
    <tr>
      <td><strong>Steppers</strong></td>
      <td>画面中央/上部</td>
      <td>手順</td>
      <td>順序が決まったプロセス（登録、決済等）</td>
    </tr>
    <tr>
      <td><strong>BackTop</strong></td>
      <td>画面下端</td>
      <td>全レベル</td>
      <td>長いページでのトップへのクイック復帰</td>
    </tr>
  </tbody>
</table>

## カテゴリ別の使い分け

<table>
  <thead>
    <tr>
      <th>カテゴリ</th>
      <th>コンポーネント</th>
      <th>推奨される利用シーン</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td rowspan="2"><strong>グローバルナビゲーション</strong></td>
      <td><strong>Navbar</strong></td>
      <td>主要な機能が少なく、横幅を広く使いたいとき。</td>
    </tr>
    <tr>
      <td><strong>Sidebar</strong></td>
      <td>機能数が多く、グルーピングが必要な管理システム。</td>
    </tr>
    <tr>
      <td rowspan="2"><strong>コンテンツの切り替え</strong></td>
      <td><strong>Tab</strong></td>
      <td>同一ページ内で関連するコンテンツを「表示領域を共有して」切り替える。</td>
    </tr>
    <tr>
      <td><strong>Segmented Control</strong></td>
      <td>表示の「ビュー」や「モード」を切り替える。</td>
    </tr>
  </tbody>
</table>

## ユーザビリティのヒント
<table>
  <thead>
    <tr>
      <th>項目</th>

    </tr>
