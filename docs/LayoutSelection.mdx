# レイアウトコンポーネントの使い分け

wimui は、柔軟な位置調整から厳密なグリッドシステムまで、複数のレイアウト手法を提供しています。

## レイアウト手法の比較

<table>
  <thead>
    <tr>
      <th>コンポーネント</th>
      <th>ベース技術</th>
      <th>主な用途</th>
      <th>使い分けのポイント</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>
        <strong>Stack</strong>
      </td>
      <td>Flexbox</td>
      <td>直線的な並び</td>
      <td>要素を一定の間隔（gap）で垂直または水平に並べる場合に最適。</td>
    </tr>
    <tr>
      <td>
        <strong>Flex</strong>
      </td>
      <td>Flexbox</td>
      <td>柔軟な配置</td>
      <td>詳細な `justify-content` 等の制御が必要な場合。</td>
    </tr>
    <tr>
      <td>
        <strong>Group</strong>
      </td>
      <td>Flexbox</td>
      <td>要素のグループ化</td>
      <td>折り返しが必要な要素の並びに最適。</td>
    </tr>
    <tr>
      <td>
        <strong>Grid</strong>
      </td>
      <td>CSS Grid</td>
      <td>多次元的な配置</td>
      <td>行と列の両方を管理する場合。</td>
    </tr>
    <tr>
      <td>
        <strong>SimpleGrid</strong>
      </td>
      <td>CSS Grid</td>
      <td>シンプルなグリッド</td>
      <td>レスポンシブなカラムレイアウトを組む場合。</td>
    </tr>
    <tr>
      <td>
        <strong>Box</strong>
      </td>
      <td>CSS</td>
      <td>基本コンテナ</td>
      <td>
        余白、サイズ、境界線などの基本的なスタイリングを付与するための最小単位。
      </td>
    </tr>
    <tr>
      <td>
        <strong>Center</strong>
      </td>
      <td>Flexbox</td>
      <td>中央揃え</td>
      <td>中身を上下左右完全に中央に配置したい場合のショートカット。</td>
    </tr>
    <tr>
      <td>
        <strong>Aspect Ratio</strong>
      </td>
      <td>CSS</td>
      <td>比率の固定</td>
      <td>画像やビデオなど、特定の縦横比を維持する必要がある場合。</td>
    </tr>
    <tr>
      <td>
        <strong>Container</strong>
      </td>
      <td>CSS</td>
      <td>最大幅の制御</td>
      <td>
        コンテンツの最大幅を制限し、大きな画面での視認性を確保する外側の枠。
      </td>
    </tr>
  </tbody>
</table>

## 高度なグリッド・レイアウト

<table>
  <thead>
    <tr>

      <th>ターゲット</th>
      <th>特徴</th>
    </tr>

    </tr>

        <strong>Splitter</strong>
      </td>
      <td>リサイズ可能な分割レイアウト</td>
      <td>

      </td>
    </tr>
  </tbody>
</table>

## レイアウト設計のベストプラクティス

<table>
  <thead>
    <tr>
      <th>項目</th>
      <th>ガイドライン</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>
        <strong>疎結合な余白</strong>
      </td>
      <td>
        コンポーネント自体に外側の余白（margin）を持たせず、`Stack` や `Group`
        の `gap` プロパティを使ってコンポーネント間の距離を制御してください。
      </td>
    </tr>
    <tr>
      <td>
        <strong>レスポンシブ設計</strong>
      </td>
      <td>
        `SimpleGrid` の `cols`
        プロパティなどを活用し、画面サイズに応じたカラム数の変化を宣言的に記述してください。
      </td>
    </tr>
    <tr>
      <td>
        <strong>セマンティックなラップ</strong>
      </td>
      <td>
        可能な限り `Box` の `as` プロップを使用して、`section`, `article`,
        `aside` などの適切な HTML タグを使用して構造化してください。
      </td>
    </tr>
    <tr>
      <td>
        <strong>コンテナによる制御</strong>
      </td>
      <td>
        コンテンツの最大幅を制限する場合は `Container`
        コンポーネントを最外殻に使用し、大きな画面での視認性を確保してください。
      </td>
    </tr>
  </tbody>
</table>
