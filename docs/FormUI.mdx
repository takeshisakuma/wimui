# 最適なフォームUIのデザイン (Form UI Best Practices)

<p
  style={{
    margin: "0 0 16px 0",
    fontSize: "18px",
    lineHeight: "1.6",
    color: "#6c757d",
  }}
>
  ユーザーが情報を入力しやすく、ストレスなく完了できるフォームを設計するためのガイドラインです。
  <br />
  入力の間違いやエラーを防ぎ、心理的な負担を軽減することで、コンバージョン率の向上を目指します。
</p>

## フォームをデザインする際の注意点

複雑さを避け、ユーザーが入力作業に迷わないように工夫することが求められます。以下の要因はユーザーの離脱を招きます。

<table>
  <thead>
    <tr>
      <th>注意点</th>
      <th>具体的な問題とユーザーへの影響</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>
        <strong>フォームが複雑で長すぎる</strong>
      </td>
      <td>
        過剰な入力項目や関連のない情報を求めると、ユーザーは負担を感じて離脱します。
        <br />
        一度に大量の選択肢を表示すると、どれが必須か分からなくなり、誤入力も増えます。
      </td>
    </tr>
    <tr>
      <td>
        <strong>
          不明瞭な指示と
          <br />
          エラーメッセージ
        </strong>
      </td>
      <td>
        指示が曖昧だったり、エラーメッセージが具体的でないと、ユーザーは混乱します。
        <br />
        修正箇所が分からないとフラストレーションが溜まり、フォーム放棄の原因となります。
      </td>
    </tr>
    <tr>
      <td>
        <strong>
          モバイル端末への
          <br />
          対応不足
        </strong>
      </td>
      <td>
        ボタンや入力欄が小さすぎると、誤タップや操作ミスを誘発します。
        <br />
        レスポンシブ対応が不十分だと、ユーザーは入力自体を諦めてしまいます。
      </td>
    </tr>
  </tbody>
</table>

## UI/UXを考慮したベストプラクティス

### ユーザーの心理的負荷を軽減する

<table>
  <thead>
    <tr>
      <th>施策</th>
      <th>内容</th>
      <th>目的・効果</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>
        <strong>ラベルの上部配置</strong>
        <br />
        (推奨)
      </td>
      <td>
        ラベルをフィールドの上に配置（トップアライン）します。
        <br />
        （詳細は <a href="/docs/guides-form-labels--docs">Form Labels</a>{" "}
        を参照）
      </td>
      <td>
        視線の移動を減らし、視認性を高めることで、入力スピードを向上させます。
      </td>
    </tr>
    <tr>
      <td>
        <strong>必須・任意項目の区別</strong>
      </td>
      <td>
        必須項目には「必須」マークを、任意項目にはラベルに「（任意）」と明記します。
      </td>
      <td>ユーザーが何を必ず入力すべきかを一目で判断できるようにします。</td>
    </tr>
    <tr>
      <td>
        <strong>適切な余白の確保</strong>
      </td>
      <td>グループ内は 4~8px、項目間は 16~24px の余白を設定します。</td>
      <td>
        Label、Input、Help Message の関連性を明確にし、情報の整理を助けます。
      </td>
    </tr>
    <tr>
      <td>
        <strong>関連情報のグループ化</strong>
      </td>
      <td>個人情報、配送先など、関連する項目をまとめて配置します。</td>
      <td>情報のまとまりを認識しやすくし、入力の流れをスムーズにします。</td>
    </tr>
    <tr>
      <td>
        <strong>入力欄の適切な幅設定</strong>
      </td>
      <td>郵便番号や住所など、入力内容の長さに合わせた幅を設定します。</td>
      <td>
        視覚的なバランスを整え、何を入力すべきかのヒント（アフォーダンス）を与えます。
      </td>
    </tr>
    <tr>
      <td>
        <strong>項目数の最適化</strong>
      </td>
      <td>
        選択肢が少ない場合はラジオボタンを使い、項目自体は必要不可欠なものに絞ります。
      </td>
      <td>クリック数を減らし、ユーザーの「面倒くささ」を軽減します。</td>
    </tr>
    <tr>
      <td>
        <strong>ステップの可視化</strong>
      </td>
      <td>複数ページにわたる場合は、進行状況バー（Stepper）を表示します。</td>
      <td>
        ゴールまでの道のりを示し、ユーザーに安心感を与えて完了率を高めます。
      </td>
    </tr>
  </tbody>
</table>

### 誤入力を防止する

<table>
  <thead>
    <tr>
      <th>施策</th>
      <th>内容</th>
      <th>目的・効果</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>
        <strong>プレースホルダーとラベル</strong>
      </td>
      <td>
        プレースホルダーは入力例（ヒント）として使い、ラベルの代わりにはしません。
      </td>
      <td>
        ユーザーが何を入力すべきか常に理解できるようにし、アクセシビリティを確保します。
      </td>
    </tr>
    <tr>
      <td>
        <strong>ヘルプテキスト</strong>
      </td>
      <td>
        重要な情報はフィールドの近くに、補足情報はツールチップで表示します。パスワードの条件などを事前に明示します。
      </td>
      <td>入力ルールを明確にし、迷いやエラーを減らします。</td>
    </tr>
    <tr>
      <td>
        <strong>文字数・フォーマットの明示</strong>
      </td>
      <td>
        「100文字以内」などの制限を事前に伝え、リアルタイムで文字数カウントを表示します。
      </td>
      <td>入力後の修正作業を防ぎ、ユーザーに安心感を与えます。</td>
    </tr>
    <tr>
      <td>
        <strong>自動変換</strong>
      </td>
      <td>
        全角英数を半角に自動変換するなど、システム側で表記ゆれを吸収します。
      </td>
      <td>ユーザーの入力負担を減らし、データ形式の不整合を防ぎます。</td>
    </tr>
    <tr>
      <td>
        <strong>オートコンプリート</strong>
      </td>
      <td>住所やメールアドレスの自動補完を有効にします。</td>
      <td>入力の手間を省略し、タイプミスを大幅に削減します。</td>
    </tr>
    <tr>
      <td>
        <strong>最適なキーパッド</strong>
      </td>
      <td>
        モバイルでは、電話番号欄には数値キーパッドを表示するなど、
        <code>input type</code> を適切に設定します。
      </td>
      <td>モバイルユーザーの入力ストレスを軽減します。</td>
    </tr>
    <tr>
      <td>
        <strong>入力方式の工夫</strong>
      </td>
      <td>
        日付はカレンダーピッカー、選択肢はドロップダウンなど、自由入力を減らします。
      </td>
      <td>フォーマットエラーを未然に防ぎ、データの一貫性を保ちます。</td>
    </tr>
    <tr>
      <td>
        <strong>
          リアルタイムバリデーション
          <br />& インラインエラー
        </strong>
      </td>
      <td>
        入力直後に検証し、該当箇所のすぐ近くにエラーメッセージを表示します。
      </td>
      <td>エラーに即座に気づかせ、その場での修正を促します。</td>
    </tr>
    <tr>
      <td>
        <strong>ボタンのアクティブ化</strong>
      </td>
      <td>
        必須項目が埋まるまで送信ボタンを非活性にする、またはクリック時に検証を行います。
      </td>
      <td>未完了状態での誤送信を防ぎます。</td>
    </tr>
  </tbody>
</table>

### ユーザー離脱を防止する

<table>
  <thead>
    <tr>
      <th>施策</th>
      <th>内容</th>
      <th>目的・効果</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>
        <strong>明確なCTA</strong>
        <br />
        (Call To Action)
      </td>
      <td>
        「送信」「次へ」など、具体的で分かりやすい文言を採用し、視覚的に目立つデザインにします。
      </td>
      <td>ユーザーが迷わず次のアクションを実行できるようになります。</td>
    </tr>
    <tr>
      <td>
        <strong>ローディング表示</strong>
      </td>
      <td>送信処理中はローディングアニメーションを表示します。</td>
      <td>処理が進んでいることを伝え、ユーザーに安心感を与えます。</td>
    </tr>
    <tr>
      <td>
        <strong>不要な外部リンクの排除</strong>
      </td>
      <td>
        入力中に外部ページへ遷移させないようにします。どうしても必要な場合は別タブで開きます。
      </td>
      <td>入力作業への集中を維持し、ページからの離脱を防ぎます。</td>
    </tr>
  </tbody>
</table>
