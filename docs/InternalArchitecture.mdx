### FieldTemplate

`FieldTemplate` provides a consistent structure for form fields, handling:
- **Label**: Automatic integration with `Label` component.
- **Error**: Integrated `FieldError` display when an error message is provided.
- **Required**: Visual indication of required fields.
- **Layout**: Support for vertical (default) and horizontal layouts.

### InputBase

`InputBase` is the visual shell for all text-style inputs. It handles:
- **Icons**: Named slots for `leftIcon` and `rightIcons`.
- **Clear Button**: Automatic display of a clear icon when `allowClear` and `hasValue` are true.
- **Status**: Visual feedback for `error` and `disabled` states.
- **Width**: Flexible width control (XS to XL or custom values).

### OverlayBase

`OverlayBase` is the foundation for floating UI elements like Dialogs and Drawers. It manages:
- **Portals**: Ensures the component renders outside the main DOM tree to avoid z-index issues.
- **Backdrop**: Handles the darkened background (Overlay) and its click-to-close behavior.
- **Focus Management**: Integrated with `FocusTrap` to prevent focus from escaping the modal.
- **Transistions**: Provides standardized entry/exit animations.
- **Performance**: Uses `will-change: transform` for high-frequency animations (like `Marquee` and `Carousel`) to ensure 60fps performance by offloading to the GPU.

## Performance Tips

### âšˇ Animation Optimization (`will-change`)
For components that involve continuous or complex animations, we use the `will-change` CSS property. This hints to the browser that an element will be animated, allowing it to optimize ahead of time.
- **Used in**: `Marquee`, `Carousel`, `Skeleton`, `Icon` (loading state), `BottomSheet`, `Tooltip`, `Switch`, `Sidebar`, `Accordion`.
- **Micro-interactions**: We also apply this to small but frequent interactions like `Switch` toggles and `Accordion` expansions to ensure zero-lag feedback.
- **Note**: Use sparingly, as it consumes memory and GPU resources. Avoid over-using on many small elements simultaneously.

In the `Image` component, we've introduced a `priority` prop. When set to `true`, it applies `fetchpriority="high"` to the underlying `img` tag.
- **When to use**: For the largest image above the fold (LCP - Largest Contentful Paint).
- **Benefit**: Tells the browser to prioritize this image over other non-critical resources.
