# フィードバック・通知コンポーネントの使い分け

システムの状態変化やユーザーの操作結果を伝える際、メッセージの重要度や持続性に応じてコンポーネントを選択します。

## コンポーネント比較

<table>
  <thead>
    <tr>
      <th>コンポーネント</th>
      <th>表示場所</th>
      <th>持続性</th>
      <th>強制力</th>
      <th>主な用途</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>Alert</strong></td>
      <td>コンテンツ内</td>
      <td>永続的/手動</td>
      <td>低</td>
      <td>画面全体やセクションに関連する重要な情報。</td>
    </tr>
    <tr>
      <td><strong>Toast / Snackbar</strong></td>
      <td>画面の端</td>
      <td>一時的 (自動)</td>
      <td>低</td>
      <td>操作の成功確認、軽微な通知。</td>
    </tr>
    <tr>
      <td><strong>Banner</strong></td>
      <td>画面上部</td>
      <td>永続的</td>
      <td>中</td>
      <td>メンテナンス予告、システム全体の重要な案内。</td>
    </tr>
    <tr>
      <td><strong>Notification</strong></td>
      <td>画面の端</td>
      <td>一時的 / 永続的</td>
      <td>中</td>
      <td>画像やアクションを含むリッチな通知。</td>
    </tr>
    <tr>
      <td><strong>Popconfirm</strong></td>
      <td>要素の付近</td>
      <td>手動で閉じる</td>
      <td>中</td>
      <td>削除などの重要な操作前の最終確認。</td>
    </tr>
    <tr>
      <td><strong>Result</strong></td>
      <td>コンテンツ中央</td>
      <td>永続的</td>
      <td>高</td>
      <td>支払い完了、404エラーなどの処理結果表示。</td>
    </tr>
    <tr>
      <td><strong>Loader / Spinner / Progress</strong></td>
      <td>コンテンツ内</td>
      <td>処理中のみ</td>
      <td>低</td>
      <td>データの読み込み状況や進行状況の可視化。</td>
    </tr>
  </tbody>
</table>

---

## ステータスの意味合い

フィードバックコンポーネントは、色のセマンティクスを活用して情報を伝えます。

<table>
  <thead>
    <tr>
      <th>ステータス</th>
      <th>カラー</th>
      <th>意味合い</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>Success</strong></td>
      <td>Green</td>
      <td>操作が成功した、正常に完了した。</td>
    </tr>
    <tr>
      <td><strong>Info</strong></td>
      <td>Blue</td>
      <td>補足的な情報、中立的な通知。</td>
    </tr>
    <tr>
      <td><strong>Warning</strong></td>
      <td>Yellow</td>
      <td>注意が必要。</td>
    </tr>
    <tr>
      <td><strong>Error</strong></td>
      <td>Red</td>
      <td>致命的な問題、操作の拒否。</td>
    </tr>
  </tbody>
</table>
