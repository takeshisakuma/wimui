# ピッカーとスライダーの使い分け

数値、日付、色、時間など、ユーザーが複雑な値を選択するための最適な UI を選択します。

## 種類別の使い分け

<table>
  <thead>
    <tr>
      <th>コンポーネント</th>
      <th>入力内容</th>
      <th>推奨される利用シーン</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>Slider</strong></td>
      <td>数値（単一）</td>
      <td>音量、明るさなどの直感的な数値調整。正確な入力よりも変化の度合いを重視する場合。</td>
    </tr>
    <tr>
      <td><strong>Range Slider</strong></td>
      <td>数値（範囲）</td>
      <td>価格帯、年齢層、時間範囲の指定。上限と下限を同時に設定する場合。</td>
    </tr>
    <tr>
      <td><strong>DatePicker</strong></td>
      <td>日付</td>
      <td>予約日、誕生日、期限の指定。カレンダー形式での視覚的な選択が必要な場合。</td>
    </tr>
    <tr>
      <td><strong>Range Calendar</strong></td>
      <td>日付（範囲）</td>
      <td>宿泊期間、データ集計期間の指定。開始日と終了日の関係性を視覚的に確認しながら選択する場合。</td>
    </tr>
    <tr>
      <td><strong>TimePicker</strong></td>
      <td>時刻</td>
      <td>アポイントメントの時間、営業時間の設定。時・分を個別に選択する場合。</td>
    </tr>
    <tr>
      <td><strong>ColorPicker</strong></td>
      <td>色</td>
      <td>デザイン調整、テーマ変更。パレットやグラデーションから直感的に色を選択する場合。</td>
    </tr>
    <tr>
      <td><strong>ColorInput</strong></td>
      <td>色 (テキスト入力)</td>
      <td>HEX / RGB 等の色コードを直接入力・貼り付けしたい場合。コピペやコード値を把握しているユーザー向け。</td>
    </tr>
    <tr>
      <td><strong>DateRangePicker</strong></td>
      <td>日付（範囲・インライン入力）</td>
      <td>開始日と終了日を2つの入力フィールドで指定したい場合。カレンダーを開かずに日付をキーボードで入力する用途に最適。</td>
    </tr>
  </tbody>
</table>

## 選択の判断基準

### 数値入力か、スライダーか？

<table>
  <thead>
    <tr>
      <th>優先事項</th>
      <th>推奨コンポーネント</th>
      <th>具体例</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>正確性が重要</strong></td>
      <td>`NumberInput`</td>
      <td>振込金額、寸法</td>
    </tr>
    <tr>
      <td><strong>直感的な操作が優先</strong></td>
      <td>`Slider`</td>
      <td>ボリューム、フィルタのしきい値</td>
    </tr>
  </tbody>
</table>

### 入力フィールドか、ピッカーか？

<table>
  <thead>
    <tr>
      <th>ユーザーの行動</th>
      <th>推奨されるアプローチ</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>値を暗記している・コピペしたい</strong></td>
      <td>`ColorInput` や直接のテキスト入力を許容する。</td>
    </tr>
    <tr>
      <td><strong>選択肢を提示し入力を補助したい</strong></td>
      <td>Overlays を伴う `DatePicker` 等のピッカーを使用する。</td>
    </tr>
  </tbody>
</table>

## ユーザビリティのヒント

<table>
  <thead>
    <tr>
      <th>項目</th>
      <th>ガイドライン</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>プレビュー</strong></td>
      <td>`ColorInput` のように、選択された値を常に視覚的にフィードバックしてください。</td>
    </tr>
    <tr>
      <td><strong>アクセシビリティ</strong></td>
      <td>キーボードユーザーのために、`Tab` でピッカー内を移動し、`Enter` で決定できることを確認してください（wimui のピッカーは標準で対応しています）。</td>
    </tr>
    <tr>
      <td><strong>無効な状態の明示</strong></td>
      <td>選択不可能な日付（過去日など）はカレンダー上でグレーアウトし、直感的に判別できるように設定してください。</td>
    </tr>
    <tr>
      <td><strong>レスポンシブ</strong></td>
      <td>モバイルデバイスでは、画面を占有しすぎないよう `Dialog` や `BottomSheet` 内にピッカーを配置することを検討してください。</td>
    </tr>
  </tbody>
</table>
