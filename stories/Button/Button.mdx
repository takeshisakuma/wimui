<h2>考慮点</h2>
<table>
  <tr>
    <td>視認性</td>
    <td>目立ちやすくして、ユーザーがすぐに見つけられるようにする<br/>
        背景色やテキスト色のコントラストを高くする</td>
  </tr>

  <tr>
    <td>クリック領域</td>
    <td>クリック領域は十分に広く設定し、誤クリックを防ぐ<br/>
        特にモバイルデバイスでは指での操作を考慮する</td>
  </tr>

  <tr>
    <td>ラベル</td>
    <td>簡潔でわかりやすい言葉を使い、次に何が起こるかを伝える<br/>
        例：送信、確認、キャンセル</td>
  </tr>

  <tr>
    <td>フィードバック</td>
    <td>クリックした際のフィードバックを提供することで、ユーザーが正しい操作を行ったことを確認できるようにする<br/>
        色の変化やアニメーションなどを利用する</td>
  </tr>

  <tr>
    <td>一貫性</td>
    <td>同じ種類のボタンは、一貫したデザインを保つことでユーザーの混乱を避ける<br/>
        サイト全体で同じスタイルを使用することが望ましい</td>
  </tr>


</table>

<h2>バリエーション</h2>
バリエーションの種類で高さがかわらないようにする
<table>
  <tr>
    <td>プライマリーボタン</td>
    <td>主要なアクションを示すボタンで、他のボタンよりも目立つデザインにする<br/>
    ユーザーに次にやってほしいアクションを示す<br/>
    画面で最もさせたいアクション</td>
    <td>「購入する」」「登録する」など</td>
  </tr>
  <tr>
    <td>セカンダリーボタン</td>
    <td>補助的なアクションを示すボタンで、プライマリーボタンよりも控えめなデザインにする</td>
    <td>「キャンセル(プライマリと対になるもの、ダイアログ、重要な変更)」や「戻る」など</td>
  </tr>
  <tr>
    <td>ターシエリボタン</td>
    <td>背景をつけないと、プライマリやセカンダリと並べた時に崩れているように見える<br/>
    背景があると押せる範囲がわかる<br/>
    「テキストリンク」や、ホバー時のみ背景が出る「ゴーストボタン」のような形式</td>
    <td>「編集」「キャンセル(重要でない変更)」「サブメニューへのリンク」「ヘルプ」「SNS共有」「新規追加」「写真のアップロード」「詳細を見る」など</td>
  </tr>
</table>

<h2>状態</h2>
意図的な演出以外で、状態の変化によってサイズが変わらないようにする(レイアウトシフトを避ける)

<table>
  <tr>
    <td>通常状態</td>
    <td></td>
  </tr>

    押せることを伝える</td>
  </tr>
  <tr>
    <td>アクティブ状態</td>
    <td>押したときの挙動<br/>
    押していることを伝える</td>
  </tr>
  <tr>
    <td>無効状態</td>
    <td>操作できなくする<br/>
    押せないことを伝える</td>

  </tr>

</table>

<h2>バリエーションX意味</h2>

<table>
  <tr>
    <td>destructiveのプライマリ</td>
    <td>特に危険な操作（アカウント削除、全データ消去など）</td>
    <td>最も目立ち、ユーザーに「本当にいいですか？」と強く警告する。</td>
  </tr>
  <tr>
    <td>destructiveのセカンダリ</td>
    <td>一般的な削除（リストから1つ消す、メッセージを消すなど）</td>
    <td>画面内で主張しすぎず、かつ「危険な操作」であることを示す<br/>
    枠線の太さ(1pxなど)の分だけ中身が小さくなるように計算</td>
  </tr>
  <tr>
    <td>destructiveのターシエリ</td>
    <td>控えめな削除（設定画面の隅にある削除、補助的な消去）</td>
    <td>他の選択肢を邪魔したくないが、見つけたときには注意を促したい場合。</td>
  </tr>
</table>
