# ボタンコンポーネントの使い分け

ユーザーのアクションを促すボタンは、その重要度や役割に応じて視覚的な重み付けを使い分ける必要があります。

## Link (a) vs Button (button)

セマンティクス（意味付け）とアクセシビリティを正しく保つために、見た目ではなく「目的」で使い分けます。

| コンポーネント | 目的 | 典型的なアクション |
| :--- | :--- | :--- |
| **Link (a)** | **ナビゲーション** | 他のページへの遷移、ページ内リンク、外部サイトへの移動 |
| **Button (button)** | **アクション** | フォームの送信、モーダルの開閉、状態の変更、計算の実行 |

### 使い分けの基準
- **Link を選ぶ基準**:
  - URLが変わる。
  - ブラウザの「戻る」ボタンで戻ることが期待される。
  - 「新しいタブで開く」という操作が可能であってほしい。
- **Button を選ぶ基準**:
  - ページ上のデータを操作（保存・削除など）する。
  - 動作を実行するだけでURLは変わらない。
  - 右クリックや「新しいタブで開く」が意味をなさない。

### 実装上の注意
- **見た目と役割**: 「見た目はボタンだが機能はリンク」という場合は、`LinkButton` コンポーネントを使用してください。これは内部的に `<a>` タグを使用しつつ、ボタンのスタイリングを適用します。
- **無効状態**: `<a>` タグには本来 `disabled` 属性が存在しません。リンクを無効化する必要がある場合は、リンクを表示しないか、意味的にボタンであるべきではないか再検討してください。

## 特殊なボタン

### 1. Fab (Floating Action Button)
画面の右下などに浮いているボタンです。
- **いつ使うか**: 画面内のどこからでもアクセスしたい、そのページで最も頻繁に行われるアクション。

### 2. Button Group
複数のボタンを繋げて表示するコンポーネントです。
- **いつ使うか**: 関連性の高い一連のアクションをまとめるとき（例：ソート順の切り替え）。

### 3. Copy Button
値をクリップボードにコピーするための専用ボタンです。
- **いつ使うか**: ユーザーが手動で選択してコピーするのが難しい値。

## ガイドライン
- **一貫性**: 同じアクションには同じ種類のボタンを常に使用してください。
- **配置**: 主要なボタン（Primary）は、一般的な視線移動の終点に配置します。
- **配置の競合**: 1つの画面に Primary ボタンは原則として1つだけに留めます。
