## 選択のガイドライン

### 1. Table (テーブル)
- **いつ使うか**: 大量のデータを正確に比較したいとき。ソートやフィルタリングが必要なとき。

### 2. Card (カード)
- **いつ使うか**: 各項目が画像や説明文、ボタンなど多様な要素を持つとき。

### 3. Accordion (アコーディオン)
- **いつ使うか**: 最初の画面では見出しだけ見せ、必要に応じて詳細を読ませたいとき。

### 4. DescriptionList (説明リスト)
- **いつ使うか**: ユーザープロフィールや商品スペックなど、特定の項目（キー）に対する値（詳細）を整理して見せたいとき。ラベルとデータの関係を明確にしたい場合に最適です。

---

## 情報設計のアドバイス
- **重要度の重み付け**: 最も重要なデータは視覚的に強調します。
- **空の状態 (EmptyState)**: 表示データがない場合は、次に何をすべきかを提示します。
